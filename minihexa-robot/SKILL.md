# miniHexa 六足机器人控制

通过 BLE 蓝牙无线控制，核心脚本：`scripts/ble_robot.py`

## 安装依赖

```bash
/opt/homebrew/bin/pip3 install bleak
```

## 快速上手

```python
import asyncio, sys
sys.path.insert(0, "/Users/mechforge1/.openclaw/workspace/skills/minihexa-robot")
from scripts.ble_robot import BLERobot

async def main():
    async with BLERobot() as robot:
        await robot.play("挥手")               # 触发动作组
        await robot.move("前进", duration_s=3) # 前进3秒后自动停
        print("距障碍:", robot.distance_mm, "mm")
        print("电量:", robot.battery_mv, "mV")

asyncio.run(main())
```

交互模式：`python scripts/ble_robot.py`

## 偏转纠偏算法（2026-03-17 验证通过）

**原理：** 振荡纠偏——当累积偏转达到阈值时，发一个反向旋转脉冲过度补偿，然后自然漂移回来，循环往复。

### 参数

| 参数 | 值 | 说明 |
|---|---|---|
| 触发阈值 | 5.5° | 累积偏转达到此值时触发 |
| 纠偏脉冲 | 250ms | 实际纠偏约 11-17°（略大于 2×阈值）|
| 步态间隔 | 500ms | 前进→纠偏、纠偏→前进之间的等待时间 |

### 执行流程

```python
drift_rate = <查表得到该速度的偏转率°/s>
trigger_time = 5.5 / drift_rate   # 累积到5.5°所需秒数

loop:
    1. 前进 trigger_time 秒
    2. 停止，等 500ms（步态稳定）
    3. 反向旋转 250ms（逆时针抵消右偏，或顺时针抵消左偏）
    4. 停止，等 500ms（步态稳定）
    5. 回到 1
```

### 纠偏方向对照

| 运动方向 | 自然偏转 | 纠偏旋转方向 | 纠偏命令 |
|---|---|---|---|
| 前进（vy>0）| 右/顺时针 | 逆时针 | `C\|0\|0\|1&` |
| 后退极慢（vy=-10）| 左/逆时针 | 顺时针 | `C\|0\|0\|2&` |
| 后退中速（vy=-20~-50）| 右/顺时针 | 逆时针 | `C\|0\|0\|1&` |
| 后退高速（vy=-60~-70）| 左/逆时针 | 顺时针 | `C\|0\|0\|2&` |
| 左移（vx<0）| 逆时针 | 顺时针 | `C\|0\|0\|2&` |
| 右移（vx>0）| 交替 | 根据实际偏转判断 | 视情况 |

### ⚠️ 关键注意事项

1. **步态间隔必须 ≥500ms**：前进/旋转切换之间必须停下等待，否则步态过渡干扰导致纠偏失效
2. **不能用同时移动+旋转**（如 `C|0|60|1&`）：自转速度固定，无法与位移速度匹配，效果不可控
3. **低电量影响**：电量低时关节变软，速度和偏转率都会变化，标定数据在充足电量下有效

## 预设路径（保留动作）

### 走正方形（实测最优，2026-03-17）

**参数：** vy=30，边长 50cm，无纠偏（满电状态效果最佳）  
**结论：** 满电时 vy=30 直行性极好，加纠偏反而引入额外误差，不如不加。

```python
async def walk_square(client, side_cm=50, speed_vy=30):
    """走正方形。默认边长50cm，vy=30（92mm/s）。"""
    SPEEDS = {30: 92, 60: 207}  # vy -> mm/s
    speed_mm_s = SPEEDS.get(speed_vy, speed_vy * 1.5)  # 估算
    side_mm = side_cm * 10
    duration = side_mm / speed_mm_s

    for i in range(1, 5):
        # 前进一边
        await send(client, f"C|0|{speed_vy}|0&")
        await asyncio.sleep(duration)
        await send(client, "C|0|0|0&")
        await asyncio.sleep(0.5)
        # 右转 90°
        await send(client, "C|0|0|2&")
        await asyncio.sleep(1.85)
        await send(client, "C|0|0|0&")
        await asyncio.sleep(0.5)
```

**注意事项：**
- 满电（>7800mV）时直行性好，无需纠偏
- 低电量时偏转加剧，可考虑加纠偏（每段 25cm 后逆时针 250ms）
- 转角累积误差约 ±5°/次，4次后最多 ±20°，实测通常 <10°
